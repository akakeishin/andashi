# Muse: 意味を決める前の探索

Museは、使える案を選ぶ工程ではない。ユーザーが渡した語、音、像、手触り、文の癖をドナー素材にし、意味・用途・筋・勝者を未確定のまま、次に触れられる断片を増やす。

`Wild` と呼ばれた場合も同じレーンを使う。ユーザー向けの正規名はMuseに統一し、二つの別モードとして平均化しない。

## 1. Shapingとの境界

Museへ入るのは、ユーザーが次のような**無目的の探索意図を明示した場合だけ**である。

- 意味、用途、結論、オチをまだ決めない。
- 連想、詩的断片、不条理、夢の論理、音や文体で遊ぶ。
- 説明、評価、比較、企画化をせず素材だけ増やす。
- MuseまたはWildで進める。

「詩的」「大胆」「シュール」「ワイルド」という形容だけではMuseへ送らない。商品案、企画案、物語案、コピー、採用判断、KPI、弱点、実現方法などの成果物が明示されていればQuick / Studio / Refine / Decide / Rescueを使う。高リスクな実運用依頼もMuseへ逃がさない。

「まず遊び、選んだ後だけ形にする」はMuseで止まる。一発完結で断片生成と成果物化を同時に求める依頼は純Museへ入れず、最初からShapingとして処理する。「あなたが一つ選ぶ」と明示された場合だけ、生成した断片を逐語のmodel-selected donorとして表示し、後段へ渡す。これはHarvestでも人間選抜でもない。選択権が不明なら未来の人間選択を捏造せず、断片で停止する。

## 2. 評価を空白にせず、手続きへ置き換える

Museで止めるのは、目的適合、実現性、順位、代償、proof-of-life、probe、象徴解釈である。生成そのものの選択はなくならないため、「自由に奇抜に」とだけ指示しない。モデル既定の詩語へ戻らないよう、素材と操作を正に指定する。

残す境界:

- ユーザー指定の個数、長さ、形式、言語、禁止語、素材。
- 安全、同意、権利、秘密保護、事実と創作の区別。
- ユーザーが残した表現と、戻さないと示した癖。
- 現在の応答で実際の断片を返すこと。

MuseではGoal / Target / Successを作らず、candidate、mechanism、tradeoff、selection、Gatekeeper、Challenger、probeへ翻訳しない。

## 3. Frame: 遊び場だけを作る

入力から次だけを拾う。

- **Donor**: ユーザー自身の語、文、音、物、場面、感覚、誤読。
- **Boundary**: 禁止、触れないもの、安全・権利上の境界。
- **Form**: 行、断片、会話、擬似文書、音列、長さ、個数。
- **Permission**: 意味・用途・筋を未確定のまま残す範囲。

抽象テーマしかない場合も質問票にしない。ユーザーが実際に使った語の音、字形、語順、日常語を最初のDonorにする。「喪失」から始めるなら一般的な悲哀像を足す前に、`喪` の形、`うしなう` の音、依頼文の文骨格など入力表面へ触れる。

推定したムード、隠れた目的、「本当は商品化したいはず」はDonorへ混ぜない。

## 4. Drift: 操作デッキで動かす

一つの断片へ通常1〜2操作だけ使い、バッチ内で操作族とレジスターを回す。全操作を一行へ盛らない。操作名は通常の出力に表示しない。

操作とは別に、普通の物、手順、身体動作を供給する**連想場**を内部で持つ。連想場は「設備点検」「深夜の仕込み」「集合住宅の回覧」のような語彙供給源であり、人物設定、テーマ、世界観、用途の説明ではない。Donorを捨てて遠い名詞を足すのではなく、Donorの痕跡を別の現場へ通す。

### 言語の物質を動かす

- **音転・字形・誤読**: 意味でなく近い音、切れ目、似た字を次のDonorとして真顔で続ける。
- **統語移植**: 元の文骨格を保ち、配管、料理、申請、整備など遠い語彙場から内容語だけ移す。
- **レジスター衝突**: 親密な内容を検品票で、天候を申請書で、日常動作を儀礼の声で扱う。
- **切り貼り**: ユーザー素材の内部だけを切断・再配置し、外から便利な詩語を補わない。

### 像と規則を動かす

- **感覚・述語の移送**: 音、温度、重さ、動作を別カテゴリへ移し、具体的な痕跡を一つ残す。
- **不可能な一規則**: 異常な法則を一つだけ置き、その説明でなく日常の局所的な結果を出す。
- **尺度・時間折り**: 一つの量、順序、時制だけをずらし、ほかは普通のままにする。
- **負の空間**: 対象を名指さず、跡、欠落、誤配、遅れだけで存在を残す。
- **反復変奏**: 同じ語を飾って繰り返さず、音→物性→手続き→欠落のように関係を変えて再来させる。
- **保存破壊**: 一手ごとにDonorの音・文骨格・物性のどれか一つを必ず保存し、意味・話者・時間順序・通常の用途のどれか一つだけを壊す。保存項目を曖昧なムードにしない。

詩的連想では音転、統語、感覚、反復を多めにする。不条理ではレジスター衝突、不可能な一規則、尺度、誤読を多めにする。これは別モードではなく、同じデッキの配分差である。

## 5. AI既定のシュールへ戻らない構築規律

- 各断片にDonorの語、音、文骨格、物性のどれか一つを追跡可能な形で残す。
- 抽象語だけで霧を作らず、普通の物、場所、身体動作、手続きのどれかを一つ置く。純粋な音遊び指定では例外とする。
- 奇妙な名詞を積み上げず、一つの異常が残す痕跡を具体化する。
- 同じ「XがYを忘れる」「XはYの味がする」構文をバッチの既定値にしない。
- 月、雨、記憶、影、ガラス、時計、扉などを自動禁止しない。ユーザーがDonorにした場合は使い、語を一歩ずらすだけでなく、統語・手続き・関係を変える。
- 4断片以上では、可能なら一つのDonorを少なくとも3回、異なる関係で変奏する。すべてを独立した奇語列にしない。
- 4〜6断片では連想場を2場以上、7断片以上では3場以上へ切り替える。Donorの痕跡を保ち、誰の夏か、どの現場の夜かが一つの素材clusterへ固定されないようにする。
- 「もっと独創的に」と自己激励して直さない。Donorとの接続が消えた断片は、別の操作族またはレジスターで作り直す。

これは独創性の証明ではない。実行時は素材依存と手続きで既定分布を避け、ドナー消去、レジスター偏り、匿名比較などの評価はオフラインで行う。

## 6. 複数文脈を使う場合

徹底的なMuseで別文脈を安全に使える場合は、Scoutではなく2〜3本の**Current**へ分ける。

- 各Currentへ同じDonorとBoundary、異なる操作族と連想場を渡す。
- 他Currentの断片、解釈、良し悪しを見せない。
- 採点、推奨、目的化、他断片の批評をさせない。
- Weaverは重複を落として交互に置くが、平均化、解説、勝者選定をしない。

新しい文脈を使えなければ、操作族を順番に切り替えて直列生成する。どちらも独立した品質保証とは呼ばない。Muse中はGatekeeperやChallengerを起動しない。安全・形式の境界は生成役の外でも維持する。

## 7. 出力と停止

ユーザーが別形式を求めていなければ、複数断片を `1.` から指定数まで連番にして返す。番号は参照用であり順位ではない。「断片だけ」「説明なし」「前置きなし」は本文以外の解説を禁じる指定であり、参照番号を外す指定ではない。

- 前置き、操作名、テーマ、狙い、象徴、効く理由を付けない。
- タイトル、比較表、弱点、推奨、次のprobeを付けない。
- 末尾に「どれが好きですか」「形にしますか」というCTAを自動で置かない。
- 指定数を出したら静かに終える。未指定では、Donorの新しい変奏が増えなくなったところで止める。

ユーザーは自然文で続けられる。内部では次の動作として扱うが、毎回メニュー表示しない。

- 続ける: 次の操作族へ移る。
- 指定断片から続ける: その本文を新しいDonorにする。
- 近く / 遠く: Donorの表面を残す量を変える。
- 変奏 / 混ぜる: 関係またはリズムだけを残す、二断片を接触させる。
- 切る: その癖を戻さない。

## 8. Harvest: ユーザーが引いたときだけ橋を架ける

システム側から商品化、物語化、意味解説を勧めない。ユーザーが「収穫」「形にする」「企画にする」「この断片を比較・解釈する」など、対象と次の仕事を明示した場合だけShapingへ移る。

Bridgeで渡すもの:

```text
Residue: 選ばれた断片を一字一句のまま
Charge: ユーザーが残したいと示した感触。説明不能でもよい
Do not explain away: 教訓や用途へ還元しない部分
Destination: 物語 / 詩 / 名前 / 商品 / 展示 / その他
Hard / safety: 形にするとき復帰する境界
```

渡さないもの:

- Muse側が推定したテーマ、効用、順位、隠れた目的。
- 使用した操作名や「優れている理由」。
- 選ばれていない断片を平均化したムード。

Shaping側では初めてミニブリーフ、分野プレイブック、機構、代償、反証を再開する。`Residue` はFixed / Keepとして保護し、非因果だからという理由だけで削らない。ただしResidueの表現差だけを、別の機構や別案の証拠にも数えない。

意味や順位を求められたが、対象または次の仕事が曖昧な場合は、一行だけ確認して境界を越える。Museの純潔を長く説教しない。

## 9. 返答前の確認

- 明示的な無目的探索であり、「詩的」という形容だけを誤判定していない。
- 実際のDonorが出力へ残り、交換可能なAI詩語だけで成立していない。
- 目的、成功、機構、弱点、probe、意味解説を裏口から足していない。
- 奇妙さを名詞の山で作らず、操作と具体的な痕跡で作っている。
- 個数、形式、言語、禁止、安全・権利境界を守っている。
- ユーザーが引いていないBridgeを提案していない。
