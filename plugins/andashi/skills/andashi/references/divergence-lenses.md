# 機構ポートフォリオの作り方

この資料はQuick / Studio / Refine / RescueのShaping専用である。意味や用途を決めないMuseには適用せず、`muse-protocol.md` のDonorと操作デッキを使う。

## 1. 最初の思いつきをクラスとして外へ出す

発散前に、このブリーフで人やモデルが最初に出しやすい答えを通常3〜5個だけ内部記録する。完成した候補名や鮮明な作例ではなく、「英語二語を圧縮したB2B名」「通知で再訪を促す」「寄せ書きを編集する」のような**機構・形態クラス**にする。これは候補数に含めず、rawの完成例をScoutへ渡さない。

この一覧は静的な禁止語表ではない。対象、制約、時点に応じて毎回作り直す。王道や定番をユーザーがHard条件にした場合は自動棄却せず、約束を残したまま対象固有の逸脱と下流効果を作る。定番クラスを候補へ残すときは、後述の定番テストを必ず通す。

## 2. 具体案より先にスロットを決める

発散前に3〜6本の探索軸を選び、互いに異なる機構スロットを作る。最低でも「主体」「介入時点」「主要機構」「支払うもの」のどれか二つ以上がスロット間で変わるようにする。

```text
S1: 問題が起きる前に、本人の選択肢を減らす
S2: 問題の最中に、仲間との交換を起こす
S3: 失敗直後に、損失を次回の資源へ変える
S4: 運営者でなく環境そのものへ制約を埋め込む
```

これは案ではない。案を作る前の探索地図である。スロットを先に固定すると、名前や媒体だけ違う兄弟案の量産を減らせる。

## 3. スロットを作るレンズ

### 主体

本人、仲間、家族、観客、購入者、専門家、運営者、影響を受ける第三者、環境のどこが動くか。権限と負担が誰へ移るかも変える。

### 介入時点

問題が起きる前、選択中、失敗直後、振り返り、次回準備、長期のどこへ働きかけるか。防止、回復、学習を別の賭けにする。

### 行動を変える機構

情報提供だけに偏らず、削除、交換、約束、共同作業、物理的制約、可逆性、段階解放、例外処理、フィードバックの遅延・即時化を使い分ける。

### 価値交換

得する人、払う人、評価される人、責任を負う人をずらす。金銭だけでなく時間、自由、精度、安心、地位、所属、誇り、面倒を見る。

### 制約の使い方

予算ゼロ、人手不足、短時間、狭い場所、既存素材だけ、オフライン限定を、欠点ではなく優先順位を強制するルールへ変える。

### 時間と規模

一度だけ、毎日、季節、成長段階、緊急時へずらす。1人、少人数、組織、地域へ拡大・縮小し、同じ機構が保たれるかを見る。

### 媒体と感覚

文章、音、空間、身体、物、会話、儀式、非同期体験へ移す。ただし媒体変更だけで別案にせず、行動と意味がどう変わるかまで作る。

### 社会関係

対称と非対称、協力と競争、秘密と公開、受け手と共同制作者を入れ替える。権力、同意、排除を隠さない。

### 失敗

成功を促すだけでなく、失敗を早くする、安くする、見えるようにする、次の資源へ変える。

### 隣接分野

図書館、舞台、物流、医療、教育、コミュニティ、ゲームなどから、見た目でなく機能原理を移植する。対象分野の制約に合わせて変形する。

4方向以上を探索する場合は、最低1本をブリーフ外の**運用現場**から始める。そのslotには、移植元、機能原理、対象制約への変形、最初の破綻条件を内部記録する。用語、外観、媒体だけを借りるのは遠距離移植に数えない。割り当ては探索quotaであり生存quotaではないため、適合しないslotは失敗理由を残して落とす。

### 観測同値崩し（実験レンズ）

依頼に具体的な観測事実が一つあり、原因によって採るべき行動が変わり得る場合だけ使う。一つの痕跡を説明できる2〜4個の原因仮説を**未確認の仮説**として並べ、同じ小介入に対する予測が分かれたときだけ、その識別介入からslotを作る。案を作った後のprobeを前へ移すだけではなく、探索単位を解決策から競合する因果モデルへ変える。

```text
Observed trace: 実際に示された、または確認できる具体的な痕跡
H1 / H2: 互いに異なる原因仮説。事実と書かない
Discriminating move: 同じ小介入
Prediction by hypothesis: 仮説ごとに異なる観測結果
Failure condition: 予測が分かれない / 痕跡が未確認 / 通常slotへ吸収される
```

観測のないゼロベース創作、原因がほぼ確定した依頼、Museには使わない。成功談、感動、話題化のような汎用痕跡を捏造しない。Challengerは凍結後の反証担当のままにし、このレンズで使った生成仮説と評価を混ぜない。最初はStudio / Rescueの任意slot一つに限り、通常slotでは出ない主要機構と反対予測が実出力で残るかを比較する。

## 4. 変換オペレーター

- **Remove**: 手順、選択肢、待ち時間、説明を一つ消す。
- **Reverse**: 作る側と使う側、先と後、評価者と被評価者を入れ替える。
- **Shift**: 判断や負担を別の主体・時点へ移す。
- **Expose**: 隠れた費用、進捗、依存、感情を見える形にする。
- **Constrain**: 一つの制約を強くし、優先順位を明確にする。
- **Make reversible**: 小さく試し、戻せる単位へ分割する。
- **Turn output into input**: 成果、失敗、反応を次回の素材にする。
- **Change the payer**: 費用だけでなく、時間や責任を誰が払うか変える。
- **Delay / accelerate**: 反応を熟成させる、または即時化して学習を速める。
- **Use the exception**: 例外を隠さず、例外処理を中心価値にする。
- **Preserve / break**: Hard条件、利用者の行動、素材の核のどれか一つを必ず保存し、既存の主体、価値交換、因果、提供手段のどれか一つを必ず壊す。何を保存・破壊したかを一組で記録する。

## 5. Scoutへ割り当てる

サブエージェントを使える場合は、各Scoutへブリーフと一つのスロットだけを渡す。他の候補、候補名、採点基準、先行Scoutの語彙を見せない。各Scoutは1〜2案を返し、良い案がなければ水増しせず失敗理由を返す。

使えない場合はスロットごとに直列で作るが、独立生成とは呼ばない。先の候補から表面要素を借りた場合は、同じスロットへ戻ったとみなす。

## 6. 同質化を観測してから脱出する

候補を作ったら、後述の置換テストと差異5軸を使う。置換後に一組でも統合された、または主体・介入点・主要機構・価値交換・代償の差が2軸未満のpairがあれば、まず同質化を観測済みとしてslotを再割り当てる。名称変更や「もっと独創的に」という自己激励で直さない。

一回の再割当で2候補以上が既存案へ吸収された、または再割当後も2軸未満のpairが残る場合だけ、別文脈を安全に使えるStudio / Rescueで**誤読リレー**を一度実行できる。各走者は直前の産物と安全・権利・秘密・Hard条件だけを受け取り、元ブリーフ、解答案、完成像を見ない。最後のIntegratorだけが元ブリーフへ再接続し、要件へ戻れない案を面白さで救済せず落とす。

一組の重複だけで自動実行しない。短いQuick、高リスク領域、単独文脈では誤読リレーを使わず、最頻回答クラス、遠距離移植、slotの再編へ戻る。これは生成操作であり、独立評価や品質保証ではない。

## 7. スロットと候補の検査

### 置換テスト

固有名詞、形容詞、媒体名を一般語へ置き換える。置換後に同じ文になるなら一つの候補へ統合する。

### 因果テスト

目立つ要素ごとに、直接の効果と次の反作用を答える。二段目がない要素は装飾の可能性が高い。

### 定番テスト

定番を使う場合は、既知パターン、対象固有の逸脱、二つ以上の下流効果を示す。定番語を使ったことを隠して独自性と主張しない。

### 飽和テスト

新しい候補が、新しい主体、介入点、機構、価値交換、代償を一つも増やさないなら発散を止める。固定12案や固定比率を埋めるために弱い案を足さない。
