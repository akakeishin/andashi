## 1. 分野の選び方

分野を選ぶのはShapingだけである。用途や意味を決めないMuseは、物語、商品、一般のどれにも分類せず、`muse-protocol.md` のDonorと操作デッキで進める。ユーザーがHarvestして成果物を指定した時点で初めて主分野を決める。

ユーザーが最終的に意思決定する成果物で主分野を決める。たとえば「新サービスの名前」は主分野をネーミング、サービスの価値提案が未確定なら商品・サービスを先に扱う。前提が未確定なのに下流の案へ進まない。

どの分野でも、次の共通骨格を保つ。

> 対象 → 現在の状況 → 働きかける機構 → 起きる変化 → 代償・反作用 → 試し方

分野名を当てること自体を目的にしない。既存分類へ無理に押し込むより、依頼固有の成功条件と失敗条件を使う。

4方向以上のポートフォリオでは、ブリーフが固定していない**知覚軸**を最低2軸変える。これは機構差に加える観察であり、見た目や語調だけ違う同一機構を救済しない。固定された文字種、媒体、時代、会場形式などは守り、残る軸で分散する。

## 2. 商品・サービス・機能

### ブリーフで拾う

- 利用者、利用場面、現在の代替手段
- 解くべき不便、欲求、進歩と、その根拠の確かさ
- 購入者、利用者、影響を受ける人の違い
- 導入、学習、継続、離脱の障壁
- 予算、期間、技術、データ、運用、法務、アクセシビリティ
- 成功指標と、失敗を早く知る指標

### 発散する軸

介入する時点、提供形態、利用者の行動、価値交換、所有者、頻度、人的支援の量、既存資産の使い方を変える。機能を足すだけでなく、手順を消す、責任を移す、判断を早める、失敗を安くする案も出す。

知覚軸は、触れる接点、操作のリズム、状態が見える／見えない範囲を変える。

### Hard gate

- 問題や需要を未検証の事実として断定しない。
- 「AI」「ダッシュボード」「通知」「ゲーミフィケーション」を価値そのものにしない。
- 誰がなぜ導入し、どの行動が変わるかを説明する。
- データ、プライバシー、安全、運用責任、アクセシビリティの重大な依存を隠さない。
- 最小の試験で中核仮説を反証できるようにする。

### 最終案に足す

対象と場面、現在の代替、価値提案、行動を変える機構、最小の入口、導入障壁、主要リスク、最初の検証を示す。物理商品なら原価帯、製造、安全、保管、配送の仮定も示す。

## 3. 企画・コンテンツ・キャンペーン

### ブリーフで拾う

- 対象者がすでに感じている緊張、関心、誤解
- 認知、理解、参加、購入、継続のどれを変えるか
- 配信チャネル、尺、頻度、制作体制、ブランド制約
- 単発の成果と、繰り返し使える企画資産
- CTA、計測方法、炎上・権利・出演者負荷

### 発散する軸

語り手、参加の強度、情報の見せる順番、ライブ性、連載単位、ユーザーが残すもの、オフラインとの接続、再利用方法を変える。

知覚軸は、語り手の立場、受け手の姿勢、反復単位を変える。

### Hard gate

- 「舞台裏」「UGC」「インフルエンサー」「バズ」を、それだけで案にしない。
- フックと目的行動の因果をつなぐ。
- チャネル固有の利用文脈と制作負荷に合わせる。
- 一回きりの話題性と、反復できる企画エンジンを区別する。
- 権利、個人情報、ブランド毀損、モデレーションの主要リスクを示す。

### 最終案に足す

対象者の緊張、企画の一文、参加または視聴の機構、代表回、継続エンジン、CTA、制作負荷、失敗指標を示す。

## 4. ネーミング・コピー

### ブリーフで拾う

- 名づける対象、カテゴリ、利用場面、対象者
- 与えたい印象と避けたい印象
- 日本語・英語などの使用言語、読み、文字数、表記制約
- 説明的、連想的、造語、人物的など許容する戦略
- 単体名か、シリーズへ展開する命名体系か
- 商標、ドメイン、SNS名、既存競合の調査有無

### 発散する手順

最初に3〜5個の意味領域を作り、各領域で異なる言語機構を使う。その後に候補名を作る。全候補を同じ接頭辞・接尾辞の量産にしない。

知覚軸は、文字種、モーラ・音節の形、語形成を変える。日英両用を「ラテン文字の英語二語合成だけ」の意味にせず、和語・漢語由来、単語、句、音象徴、翻字の可否をHard条件の範囲で検討する。`Sync / Link / Flow / Weave / Loom / Pulse / Nex`系語幹や `-exa / -ly / -io`系語尾へ一歩ずらしただけの同族を別戦略と数えない。

### Hard gate

- 音、意味、表記をユーザーの言語と利用場面で確認する。
- 読めない、聞き分けにくい、入力しにくい可能性を示す。
- 商標、ドメイン、アカウント、競合との非衝突を、調査せず断定しない。
- `AI`、`Hub`、`Plus`、`ミライ` などを使う場合は、対象固有の意味と下流効果を要求する。
- 法的な利用可否は専門調査の代わりにしない。

### 最終案に足す

命名領域、候補、読み、意味の機構、狙う印象、弱点・誤読、展開例、未確認事項を示す。既定では領域を先に見せ、同一領域内の細かな候補と領域間の戦略差を混同しない。

## 5. イベント・体験

### ブリーフで拾う

- 参加者、主催者、会場側の目的
- 開催前、到着、参加中、退出後の体験
- 人数、時間、予算、会場、スタッフ、機材、天候
- 参加の任意性、アクセシビリティ、安全、年齢、言語
- 参加者同士の関わりの強度と、持ち帰るもの

### 発散する軸

観客と参加者の境界、進行順、空間、同期・非同期、個人・小集団・全体、記録、持ち帰り、偶然性を変える。

知覚軸は、中心素材、身体が触れる／避ける関係、空間内の移動を変える。

### Hard gate

- 「基調講演＋パネル＋交流会」を目的に合う理由なく既定にしない。
- Run of show、収容、導線、音、休憩、撤収、人員を無視しない。
- 車椅子、聴覚・視覚、食物、年齢、言語など該当する配慮を入れる。
- 雨天、機材故障、欠員、参加者不足、過密の代替を示す。
- 思い出深さと安全・実行可能性を交換しない。

### 最終案に足す

参加者の変化、中心体験、簡易run of show、参加方法、必要資源、主要な失敗時対応、成果の測り方を示す。

## 6. 業務改善・問題解決

### ブリーフで拾う

- 観測された症状、基準値、困っている主体
- 根本原因の仮説と、原因だと判断した根拠
- 現在の手順、責任者、例外、統制、監査
- 変更できるもの、変えられないもの
- 移行、教育、失敗時の復帰、二重運用の負荷
- KPIと、別の場所へ負担を移していないかを見る指標

### 発散する軸

削除、順序変更、権限移譲、標準化、入口制御、バッチ化、セルフサービス、手作業の支援、例外の隔離、情報の発生源での捕捉を検討する。

知覚軸は、作業が現れる面、何が誰に見えるか、介入のリズムを変える。

### Hard gate

- 症状を原因と決めつけない。
- 「自動化」「一元化」「可視化」を、それだけで改善としない。
- 責任、法務、監査、人間の承認、ロールバックを消さない。
- 効果の基準値、測定期間、負担の移転を明示する。
- 全面導入より先に、可逆なパイロットを作る。

### 最終案に足す

原因仮説、介入点、変える手順、変えない統制、期待する機構、必要資源、失敗時の戻し方、パイロット、KPIを示す。

## 7. 物語・キャラクター・世界観

### ブリーフで拾う

- 媒体、尺、対象者、ジャンル、狙う感情や余韻
- 主人公または中心主体の欲望、恐れ、誤信、関係
- 何が選択を難しくし、どんな代償が生じるか
- 参考作品の固有要素ではなく、好きな性質
- 制作規模、表現上の制約、避けたい定番

### 発散する軸

欲望、対抗力、世界の規則、支払う価格、関係の非対称、時間、語り手、媒体固有の表現を変える。奇妙な小物ではなく、選択と結果を変える仕掛けを中心にする。

知覚軸は、時代・生活文脈、話者の立場、感覚レジスターを変える。

### Hard gate

- 固有名詞や口癖を外しても中核が残る。
- 少なくとも3つの異なる場面・展開を自然に生むエンジンがある。
- 規則、圧力、選択、結果が因果でつながる。
- キャラクターの設定が、追い詰められたときの選択へ現れる。
- 参考作品の表面要素を置き換えただけにしない。
- 媒体と制作規模で表現できる。

### 最終案に足す

一文の核、体験価値、中心欲望、対抗力、規則または関係の圧力、代償、象徴的な場面、3つの試験場面、枯れやすい弱点を示す。

旧アプリの「会話で定着するか」は最終候補の任意試験としてだけ使う。日常、圧力、親密さの3場面で同じ特徴を自然に再利用し、選択・関係・観客体験を変えない呼び名や小物は削る。

## 8. 一般・未分類

研究テーマ、学習法、贈り物、旅行の過ごし方、コミュニティ運営、制度、展示、ゲームルールなど、既存6分野へ自然に収まらない依頼に使う。

### ブリーフで拾う

- 誰が何を決めるための案か
- 現在の代替と、変えたい具体的な状態
- 成功を判断する観察可能な条件
- 使える資源、期限、不可逆性
- 影響を受ける第三者、同意、安全、維持負荷
- 事実確認が必要な前提

### 発散する軸

主体、介入時点、行動を変える機構、価値交換、制約の使い方、失敗からの回復を変える。カテゴリ固有のテンプレートを足すより、依頼固有の反証質問を作る。

知覚軸は、成果の表現、参加の形、時間の形から依頼固有に選ぶ。

### Hard gate

- 成功条件へつながらない面白さだけの案を残さない。
- 誰が実行し、誰が負担を払い、どう止めるかを隠さない。
- 未確認の制度、価格、可用性、効果、他者の意思を事実にしない。
- 高リスクまたは不可逆な案には、専門確認、同意、ロールバックを含める。
- 最も結論を変える未知を小さく確かめられるようにする。

### 最終案に足す

依頼固有の成功条件、一手、proof-of-life、影響を受ける人、最大の未知、最初のprobeを示す。定型見出しより、意思決定に必要な証拠を優先する。

## 9. 複合分野

上流から下流へ処理する。価値提案が未確定なら商品案を先に、商品案が決まってから命名やキャンペーンへ進む。複数分野の評価軸を全部足して巨大な表にせず、主分野のHard gateと、副分野の致命的な条件だけを適用する。
